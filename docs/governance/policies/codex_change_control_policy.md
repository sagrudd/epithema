# Codex Change Control Policy

## 1. Scope

This policy defines repository handling requirements for material changes made
through Codex during the EMBOSS-RS reboot effort.

## 2. Normative Rules

- After each material Codex prompt, the resulting repository changes must be
  reviewed, committed, and pushed.
- This requirement applies to documentation work, code changes, and other
  material project updates.
- The policy should be interpreted conservatively when judging whether a change
  is material.
- Local repository state should not be allowed to drift indefinitely after a
  material Codex-assisted change.
