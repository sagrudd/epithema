# Documentation and Autodoc Policy

## 1. Scope

This policy defines the mandatory documentation expectations for `emboss-rs`,
including the requirement that `autodoc` support be treated as a core project
obligation.

## 2. Normative Rules

- `autodoc` is mandatory for `emboss-rs`.
- Documentation source material should be organized in a normalized form that a
  future Sphinx build can consume without a disruptive content migration.
- Governed command metadata should be maintained in a way that supports reliable
  generated documentation.
- Documentation updates must accompany material changes to tool behavior,
  architecture, or governance.
- Manual prose and generated documentation should present the same public command
  surface and project terminology.
