# Code Structure and Module Naming Policy

## 1. Scope

This policy defines governance expectations for repository organization, module
naming, and structural discipline in `emboss-rs`.

## 2. Normative Rules

- Repository structure should reflect stable concepts rather than temporary
  experimentation.
- Module and package names should be descriptive, professional, and durable.
- Shared infrastructure, command dispatch, and family-specific implementations
  should remain clearly separated.
- Documentation naming and code naming should stay aligned where they describe
  the same governed concept.
