# EMBOSS-RS Governance Documentation

This section contains the project-governing documentation for `emboss-rs`.
Its purpose is to provide a single maintained entry point for normative policy,
with supporting policy modules and reference appendices arranged in a durable
layout that can be ingested by a future Sphinx documentation build.

## Canonical Entry Point

The canonical governance document is
[EMBOSS-RS Governance Manual](./emboss_rs_governance_manual.md). That manual is
the primary source of truth for project intent, scope, delivery expectations,
documentation policy, and repository operating rules.

## Normative Documents

The following documents are normative and should be read as project policy:

- [EMBOSS-RS Governance Manual](./emboss_rs_governance_manual.md)
- [Tool Surface and Family Policy](./policies/tool_surface_and_family_policy.md)
- [Documentation and Autodoc Policy](./policies/documentation_and_autodoc_policy.md)
- [Codex Change Control Policy](./policies/codex_change_control_policy.md)
- [Code Structure and Module Naming Policy](./policies/code_structure_and_module_naming_policy.md)

Where there is duplication for navigation purposes, the governance manual governs.

## Appendices and Reference Material

The following documents are reference material. They support implementation and
future expansion but do not supersede the normative governance documents:

- [Family-to-Tool Mapping Reference](./appendices/family_to_tool_mapping_reference.md)

## Maintenance Notes

New governance material should be added under `docs/governance/` and linked from
this index. Normative policy belongs in the governance manual and, where useful,
in the `policies/` directory. Supporting examples, registries, and lookup tables
belong in `appendices/`.
